# 코드 집중화

똑같은 코드가 여러곳에 존재한다? 비효율적

그래서 그 코드의 기능을 하나에 구현하고, 원래 코드가 있던 부분에서 이를 호출.

# 추상화

코드 집중화 = > 서비스 집중화 (캡슐 단위의 공통 서비스)

예 )

    원
    draw()
    move()
    set()

    ==================
    
    네모

    draw{}
    move()
    set()

위와같이 캡슐이 있다? 복사붙여넣기하면 나중에 수정할때 너무 머리가 아프다.

그래서 캡슐단위의 공통 기능을 집중화시키는거임.

원, 네모, 세모

=> 도형.

어떻게?  IS A 상속을 통해

위와같은 클래스를 추상클래스라고한다. (쉽게 말해 일반화한 클래스)
